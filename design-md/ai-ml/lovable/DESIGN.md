## Overview

Lovable is an AI app builder, and its homepage behaves like a product surface
before it behaves like a marketing page. The first viewport does not ask users
to read a long value proposition before acting. It shows the navigation, a
small product-news pill, the headline "Build something Lovable", the line
"Create apps and websites by chatting with AI", and a prompt composer with
"Build" and "Send message" actions. That composer is the brand's main
interaction pattern.

The current homepage verified through Lightpanda includes:
- A top navigation with Solutions, Resources, Community, Enterprise, Pricing,
  Security, Log in, and Get started.
- A "New Better SEO - Apps built to be found" announcement linking to the SEO
  page.
- A hero composer placeholder reading "Ask Lovable to create a dash".
- A trust line: "Teams from top companies build with Lovable".
- A "Meet Lovable" section with three steps: Start with an idea, Watch it come
  to life, and Refine and ship.
- A "Discover templates" section with current examples including Personal
  portfolio, Lovable slides, Architect Portfolio Website Template, Fashion blog,
  Event Platform Website Template, Personal blog, Lifestyle Blog, and Ecommerce
  Store Website Template.
- A "Lovable in numbers" proof section: 36M+ projects built, 200K+ projects
  built per day, and 300M visits per day to Lovable-built applications.
- A closing "AI App Builder" / "Ready to build?" composer repeat.
- A broad footer across Company, Product, Resources, Legal, and Community.

Lovable's design position is warm-neutral and builder-first. It rejects both
cold enterprise AI chrome and loud neon "AI magic" styling. The page reads like
a carefully made notebook for building software: cream paper, charcoal ink,
soft screenshot cards, large calm type, and tactile buttons.

**Key Characteristics:**
- Parchment cream canvas (`{colors.canvas}` - #f7f4ed) used as the default page
  floor and most component surfaces.
- Near-black charcoal (`{colors.ink}` - #1c1c1c) for headings, body text, and
  primary controls.
- Camera Plain Variable as the single brand typeface, using weight, size, and
  spacing rather than font mixing to create hierarchy.
- Opacity-driven neutrals. Most grays are charcoal or muted text at different
  opacity levels rather than unrelated neutral hex values.
- Low elevation: one-pixel warm borders and subtle focus shadows replace
  floating cards.
- Signature button treatment: dark charcoal fill with a multi-layer inset
  highlight/ring/drop shadow.
- Prompt composer as the dominant product component, repeated in hero and
  pre-footer contexts.
- Screenshot-led template cards used as proof that Lovable outputs real apps
  and websites, not just abstract AI concepts.
- Soft pulse imagery in hero/CTA zones, kept atmospheric and behind content.

## Colors

### Brand Foundation

- **Canvas** (`{colors.canvas}` - #f7f4ed): The defining Lovable surface. Use
  it for page backgrounds, hero sections, card surfaces, footer, composer
  containers, and quiet controls. It should feel like warm paper, not beige
  decoration.
- **Canvas Raised** (`{colors.canvas-raised}` - #fcfbf8): A slightly brighter
  cream-white for text on dark buttons, small raised panels, or image-backed
  surfaces that need extra contrast.
- **Ink** (`{colors.ink}` - #1c1c1c): Primary text, display headings, nav links,
  and dark CTA backgrounds. It is close to black but warmer and softer than
  `#000000`.

  high-contrast logo or brand mark moments. Most UI should stay on
  `{colors.ink}`.

## Typography

### Font Family

Lovable uses **Camera Plain Variable** as the primary and effectively single
brand typeface. The fallback stack is `ui-sans-serif, system-ui, sans-serif`.
Unlike many AI sites, Lovable does not need a separate serif display face or a
monospace-heavy product aesthetic. Its warmth comes from Camera Plain's rounded
humanist details and the cream surface system.

Use the same family for display, body, nav, buttons, captions, and composer
text. Create hierarchy through size, line-height, spacing, and a narrow weight
range.

### Hierarchy

| Token | Size | Weight | Line Height | Letter Spacing | Use |
|---|---:|---:|---:|---:|---|
| `{typography.display-xl}` | 60px | 600 | 1.05 | -1.5px | Homepage hero headline |
| `{typography.display-xl-light}` | 60px | 480 | 1 | 0 | Alternate lighter display moments |
| `{typography.display-lg}` | 48px | 600 | 1 | -1.2px | Section headings and large stats |
| `{typography.display-md}` | 36px | 600 | 1.1 | -0.9px | Subsection heads |
| `{typography.display-sm}` | 28px | 600 | 1.15 | -0.4px | Compact callouts |
| `{typography.title-lg}` | 24px | 400 | 1.25 | 0 | Larger card titles |
| `{typography.title-md}` | 20px | 400 | 1.25 | 0 | Template card titles |
| `{typography.title-sm}` | 18px | 400 | 1.35 | 0 | Step headings and small feature titles |
| `{typography.body-lg}` | 18px | 400 | 1.38 | 0 | Hero support copy |
| `{typography.body-md}` | 16px | 400 | 1.5 | 0 | Default body, footer links |
| `{typography.body-sm}` | 14px | 400 | 1.5 | 0 | Nav, captions, compact descriptions |
| `{typography.caption}` | 13px | 400 | 1.45 | 0 | Badges, metadata |

### Principles

Display typography should be confident but not loud. Use 600 as the maximum
common weight and apply negative letter-spacing only at display sizes. A 60px
hero headline can take `-1.5px`; a 48px section heading can take `-1.2px`; a
36px heading can take `-0.9px`. Body, nav, and button text should remain at
normal tracking.

Do not introduce 700/800 weights to create hierarchy. Lovable's visual system
is restrained; heavier weights make it feel like a generic startup landing
page. Use whitespace and size first.

### Note On Font Substitutes

If Camera Plain Variable is unavailable, use a warm humanist sans rather than a
geometric one. Good substitutes include Inter, Söhne, Avenir Next, or
system-ui. Avoid Helvetica/Arial for display if possible; they flatten the
brand voice and make the page feel less handcrafted.

## Layout

### Spacing System

- **Base unit:** 8px, with some 10px/12px intermediate values for Lovable's
  compact controls.
- **Small tokens:** `{spacing.xxs}` 4px, `{spacing.xs}` 8px, `{spacing.sm}`
  10px, `{spacing.md}` 12px, `{spacing.lg}` 16px.
- **Component tokens:** `{spacing.xl}` 24px, `{spacing.xxl}` 32px,
  `{spacing.xxxl}` 40px, `{spacing.block}` 56px.
- **Section tokens:** `{spacing.section-sm}` 64px, `{spacing.section-md}` 80px,
  `{spacing.section-lg}` 96px, `{spacing.section-xl}` 128px.
- **Editorial tokens:** `{spacing.editorial-lg}` 176px and
  `{spacing.editorial-xl}` 208px for large empty pauses.

The high end of the scale is important. Lovable feels calm because major
sections have room to breathe while controls themselves stay compact.

### Grid & Container

- **Max content width:** approximately 1200px centered.
- **Hero:** centered single-column stack. Announcement pill, headline, subcopy,
  composer, and trust line align to the center.
- **Meet Lovable:** three-column explanation at desktop, then two-column or
  single-column on smaller screens.
- **Template gallery:** screenshot-card grid. Use 4-up or 3-up at wide desktop
  if space allows, 2-up on tablet, and 1-up on mobile.
- **Metrics:** three equal columns on desktop. Stack on mobile with generous
  vertical spacing.
- **Footer:** multi-column utility grid, grouped by semantic category.

### Whitespace Philosophy

Lovable separates content through pacing rather than heavy dividers. The canvas
stays consistent, and the eye moves because sections have distinct density:
the hero is compact around the composer, the template section is image-dense,
the metrics section is sparse, and the footer is link-dense.

Large blank space should still feel useful. Pair it with a soft pulse image,
strong heading, or centered composer so it reads as calm focus instead of an
unfinished page.

## Elevation & Depth

| Level | Treatment | Use |
|---|---|---|
| Flat | No shadow, cream background | Page floor, hero bands, footer groups |
| Hairline | `1px {colors.border}` | Template cards, screenshots, dividers, inputs |
| Interactive hairline | `1px {colors.border-strong}` | Outline buttons, focused secondary controls |
| Tactile inset | `{elevation.button-inset}` | Primary charcoal buttons, pill icon controls |
| Focus | `{elevation.focus}` plus optional ring | Composer, inputs, keyboard focus |
| Hover soft | `0 1px 3px rgba(28,28,28,0.06)` | Rare template card lift |

The elevation philosophy is **border first, shadow rare**. Most Lovable cards
should not float. A template card gets its boundary from a warm border and a
real screenshot, not a heavy shadow.

### Button Inset

The dark button inset is the signature tactile detail:
`rgba(255,255,255,0.20) 0 0.5px 0 0 inset`,
`rgba(0,0,0,0.20) 0 0 0 0.5px inset`, and
`rgba(0,0,0,0.05) 0 1px 2px 0`.

This creates the feeling of a small crafted control pressed into the cream
surface. Use it on primary buttons, icon pills, and composer controls that need
priority. Do not apply it to every card.

### Decorative Depth

The hero and closing CTA use pulse imagery as environmental depth. It can be a
soft warm/pink/blue image, not an abstract neon orb field. The image should
stay behind content, remain low contrast, and never reduce readability of the
headline or composer.

Template screenshots add another kind of depth. They should display actual
project output and UI states, because Lovable's claim is that users can build
real apps and websites quickly.

## Shapes

### Border Radius Scale

| Token | Value | Use |
|---|---:|---|
| `{rounded.xs}` | 4px | Small internal marks, tiny menu details |
| `{rounded.sm}` | 6px | Standard buttons, inputs, menu buttons |
| `{rounded.md}` | 8px | Compact cards, tabs, secondary panels |
| `{rounded.lg}` | 12px | Template cards, screenshots, composer container |
| `{rounded.xl}` | 16px | Larger grouped containers and footer panels |
| `{rounded.pill}` | 9999px | Announcement pills, chips, mode toggles |
| `{rounded.full}` | 9999px | Circular icon buttons and avatar-like elements |

Use the radius scale deliberately. A primary "Get started" button should be a
6px rectangle, not a full pill. A "New" announcement badge should be a pill.
Template screenshots should generally use 12px so the output feels polished
without becoming playful.

### Image Treatment

Images are product proof. Use them at real aspect ratios with subtle warm
borders. Avoid heavy crops that hide the UI output. The current homepage uses
template screenshots for portfolio, slides, architecture, blog, event, personal
blog, lifestyle blog, and ecommerce examples; keep those examples visually
diverse while making the card frames consistent.

### Iconography

Lovable's page is text-and-screenshot led. Icons should support actions rather
than decorate sections. Use simple line icons for send, menu, dropdown, and
utility controls. Keep icon strokes aligned with the charcoal palette and do
not introduce colored icon sets unless the product surface requires them.

## Components

### Top Navigation

**`nav-bar`** - Cream navigation surface, about 64px tall, with Lovable
wordmark/home link at left, product/resource links across the middle, and Log
in / Get started actions at right. Nav links use `{typography.nav-link}` and
`{colors.ink}`. The "Get started" action uses `{component.button-primary}`.

On mobile, collapse secondary links behind `{component.nav-menu-button}` while
preserving a visible path to get started. The menu surface should remain cream,
not switch to a dark overlay.

### Buttons

**`button-primary`** - Charcoal CTA. Background `{colors.ink}`, text
`{colors.on-dark}`, radius `{rounded.sm}`, padding 8px x 16px, minimum height
36px, and `{elevation.button-inset}`. Use for Get started, Build, and high
priority conversion actions.

**`button-primary-large`** - Hero/composer scale version with 10px x 18px
padding and 44px minimum height. Use when the button sits inside a large prompt
or final CTA area.

**`button-primary-active`** - Same as primary with opacity reduced to 0.8.
Lovable favors opacity feedback over dramatic color changes.

**`button-secondary`** - Transparent or cream surface, charcoal text,
`1px solid rgba(28,28,28,0.40)`, radius 6px, same compact sizing as primary.
Use for Log in, Documentation, View all, or non-primary page actions.

**`button-cream`** - Quiet cream button without a strong outline. Use inside
toolbars or composer rows where too many borders would clutter the surface.

**`button-icon-pill`** - 36px circular or pill-like icon control with the inset
shadow. Use for send, microphone, mode toggles, or carousel controls when they
need tactile priority.

**`text-link`** - Charcoal text with underline. Use for inline destination
links and template URLs when the link itself is the content.

### Hero And Composer

**`hero-band`** - Centered cream section with a small announcement pill, a
large display headline, one support sentence, the composer, and a trust line.
Padding around 96px on desktop. The content should feel like a build console
embedded in an editorial homepage.

**`announcement-pill`** - Small full-pill label. Current content: "New Better
SEO - Apps built to be found". Use a very soft charcoal tint, compact caption
type, and a pill radius. The "New" part can be slightly stronger, but it should
not become a saturated badge.

**`ai-composer`** - The key component. Cream surface, warm border, 12px radius,
min height around 116px, 12px internal padding, prompt text in
`{typography.composer}`, and controls aligned at the lower edge. It should feel
like a creation field, not a search input.

**`ai-composer-focused`** - Border shifts to `{colors.border-strong}` and gains
the soft focus shadow. Keyboard focus can add the blue ring, but the visual
personality should remain warm.

**`composer-mode-toggle`** - Full pill control used for "Build" or mode-like
choices. It may use the inset shadow when active. Keep labels short.

**`composer-suggestion-pill`** - Optional idea chips or prompts. Use soft

## Do's and Don'ts

### Do

- Anchor every page on `{colors.canvas}`. The cream canvas is the brand's
  strongest visual signal.
- Use `{colors.ink}` for both type and primary button fills.
- Use Camera Plain Variable across the whole system and create hierarchy with
  size, spacing, and a narrow 400/480/600 weight range.
- Put the prompt composer early, especially on pages that invite building or
  prototyping.
- Use real screenshots or output examples to prove what Lovable can generate.
- Use warm borders for containment instead of heavy card shadows.
- Use `{elevation.button-inset}` on primary controls and tactile icon pills.
- Keep announcement pills small, calm, and close to the hero headline.
- Preserve the broad footer taxonomy on long product pages.
- Let section spacing breathe; Lovable should feel calm even when the footer is
  dense.

### Don't

- Don't use pure white as the main background.
- Don't introduce saturated blue/purple AI gradients as dominant brand color.
- Don't make the homepage read like a generic SaaS feature-card grid before
  showing the composer.
- Don't use heavy 700/800 font weights.
- Don't apply full-pill radius to normal rectangular CTAs.
- Don't use heavy shadows for cards or screenshots.
- Don't make pulse imagery dark, blurred beyond recognition, or high contrast.
- Don't hide product examples behind abstract illustration.
- Don't treat footer links as an afterthought; they are part of Lovable's
  utility-oriented system.
- Don't over-style hover states. Border, opacity, or a very soft shadow is
  enough.

## Responsive Behavior

### Breakpoints

| Name | Width | Key Changes |
|---|---:|---|
| Mobile Small | < 600px | Single-column everything, compact hero, composer full width |
| Mobile | 600-640px | Standard mobile spacing, stacked templates |
| Tablet Small | 640-768px | 2-up template grid possible, nav still compact |
| Tablet | 768-1024px | Horizontal nav can return if space allows, 2-up/3-up grids |
| Desktop | 1024-1280px | Full nav, centered hero, 3-up sections, broad footer |
| Wide | > 1280px | Max container caps near 1200px, additional outer breathing room |

### Touch Targets

- Primary and secondary buttons should be at least 36px high; hero actions can
  be 44px.
- Icon buttons should be 36px minimum, with surrounding spacing bringing the
  effective tap area closer to 44px.
- Composer controls should be easy to tap without moving the text area.
- Footer links need more vertical spacing on touch layouts than desktop link
  lists.

### Collapsing Strategy

- Top nav collapses to a compact menu below tablet widths. Keep Get started or
  the main action reachable.
- Hero headline drops from 60px to 48px and then to 36px on narrow screens.
  Reduce negative tracking proportionally.
- The composer remains the primary artifact. It should span the available width
  and retain enough height to feel like a prompt field rather than a single
  search line.
- The three Meet Lovable steps stack vertically on mobile.
- Template cards collapse from multi-column grids to a single vertical feed.
  Preserve screenshot aspect ratio and avoid clipping UI details.
- Metrics stack with clear number-label pairing.
- Footer columns become grouped accordions or stacked link groups if needed.

### Image Behavior

- Template screenshots should retain a stable aspect ratio across breakpoints.
- If a screenshot becomes too small to inspect, stack cards rather than shrinking
  the image further.
- Pulse background imagery can simplify on mobile. Reduce visual intensity
  before reducing text contrast.
- Avoid text overlays on template screenshots; titles and descriptions should
  remain outside or below the image.

## Iteration Guide

1. Begin with `{colors.canvas}` and `{colors.ink}`. Add other colors only when
   the component role requires them.
2. Identify whether the surface is an action, example, proof point, or utility
   area. Use the matching component token before inventing a new pattern.
3. Use `{component.ai-composer}` for build-oriented flows. If a page asks users
   to create something, the composer should usually be visible early.
4. Use screenshots for proof. If the section is about templates or examples,
   the card image should do most of the selling.
5. Favor borders over shadows. Add a shadow only for focus or rare hover lift.
6. Keep display text tight and large; keep UI/body text normally tracked.
7. Use two dominant type weights: 400 for most UI and 600 for display.
8. Keep CTA buttons rectangular at `{rounded.sm}` unless the control is a chip,
   badge, or icon pill.
9. Recheck mobile after every major layout change. The composer, announcement
   pill, and headline are the most likely to crowd.
10. Preserve current homepage content when cloning the page pattern: SEO
    announcement, three-step explanation, template gallery, stats, final build
    CTA, and broad footer.

## Known Gaps

- Source capture for the current homepage was performed with Lightpanda against
  `https://lovable.dev/` on 2026-05-17. It provided page text, navigation,
  links, template names, metrics, image URLs, footer groups, and cookie controls.
- Lightpanda did not expose reliable computed CSS values for every element, so
  the low-level color, typography, radius, and shadow tokens combine the current
  page structure with the existing Lovable design analysis.
- Camera Plain Variable appears to be a custom/brand typeface and may not be
  available as a public web font. Use the substitute guidance in Typography
  when implementation cannot load it.
- Exact pulse image behavior, template screenshot crop ratios, and runtime
  animations should be checked in a visual browser pass if pixel-level
  reproduction is required.
- Product-app internals are out of scope. This document describes the marketing
  and homepage design language, not the full logged-in Lovable builder UI.
- Pricing, security, enterprise, and docs pages may add additional stateful
  components that are not captured from the homepage alone.
